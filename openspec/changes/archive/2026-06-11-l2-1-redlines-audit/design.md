   ▼

## Data model

```typescript
// src/services/audit/types.ts (NEW)
export type RedLineBacking = 'cli-backed' | 'partial' | 'prose-only';

export interface RedLineEntry {
  readonly id: string;                    // e.g. 'rl-solo-code-ban-001'
  readonly rule: string;                  // human-readable rule, e.g. 'Solo Code-Change Red Line'
  readonly source: {
    readonly file: string;                // relative to project
    readonly line: number;
    readonly marker: string;              // the matched marker (MANDATORY, BLOCKING, etc.)
    readonly context: string;             // ±2 lines of surrounding context
  };
  readonly backing: RedLineBacking;
  readonly enforcerRef: string | null;    // path to the enforcement, or null if prose-only
}

export interface RedLineAudit {
  readonly totalRedLines: number;
  readonly cliBacked: number;
  readonly partial: number;
  readonly proseOnly: number;
  readonly audit: readonly RedLineEntry[];
}

export interface RedLineAuditResult {
  readonly ok: true;
  readonly command: 'audit.red-lines';
  readonly data: RedLineAudit;
  readonly warnings: readonly string[];
  readonly nextActions: readonly string[];
}
```

### 2. no-root-pollution

```typescript
// src/services/audit/enforcers/no-root-pollution.ts (NEW)
const ROOT_FILE_ALLOWLIST = new Set([
  'README.md', 'LICENSE', 'LICENSE.md', 'package.json', 'pnpm-lock.yaml',
  '.gitignore', '.gitattributes', '.editorconfig', '.npmrc', '.nvmrc',
  // directories
  'openspec', '.peaks', '.claude', 'skills', 'docs', 'tests',
  'bin', 'scripts', 'schemas', 'output-styles',
  // generated by build / publish
  'dist', 'node_modules', 'coverage',
]);

export function isRootWrite(filePath: string, projectRoot: string): boolean {
  const rel = path.relative(projectRoot, filePath);
  const top = rel.split(path.sep)[0];
  return !ROOT_FILE_ALLOWLIST.has(top);
}
```

Wired into `src/services/hooks/pre-tool-use-edit.ts` (NEW). Deny: "no-root-pollution: file `<path>` is not in the root allowlist."

### 3. sub-agent-sid

Reuses Slice 0.5 `src/services/workspace/sid-naming-guard.ts` (already shipped). New wrapper:

```typescript
// src/services/audit/enforcers/sub-agent-sid.ts (NEW)
import { isValidSessionId } from '../../workspace/sid-naming-guard.js';

export function findInvalidSids(projectRoot: string): readonly string[] {
  const subAgents = path.join(projectRoot, '.peaks/_sub_agents');
  if (!fs.existsSync(subAgents)) return [];
  return fs.readdirSync(subAgents)
    .filter((sid) => !isValidSessionId(sid));
}
```

Called by `peaks audit red-lines` and already by `peaks workspace clean` (Slice 0.5 Task 7).

### 5. mock-placement

```typescript
// src/services/audit/enforcers/mock-placement.ts (NEW)
const MOCK_PATTERNS = [
  /\bmockData\s*[:=]\s*\{/,                    // mockData: { ... } inline
  /\bfixtures?\s*=\s*\{/,                     // fixtures = { ... } inline
  /const\s+\w*[Mm]ock\w*\s*=\s*\{[\s\S]{20,}/, // const fooMock = { ... } > 20 chars
];

export function hasInlineMock(content: string): boolean {
  return MOCK_PATTERNS.some((p) => p.test(content));
}
```

Wired into `peaks slice check` as a 5th item (after the existing 4).

## Backward compatibility

- `peaks audit` is a NEW top-level — verified no existing `program.command('audit')` registration.
- `peaks request transition spec-locked` adds ONE new prerequisite check (tech-doc.md exists); existing transitions unchanged.
- `peaks slice check` adds a 5th check (mock-placement); existing 4 checks unchanged; check ordering append-only.
- `peaks workspace clean` already calls sub-agent-sid (Slice 0.5 Task 7); no change.
- `peaks doctor` unchanged.
- All hooks are NEW files; existing hook install path is unchanged (`peaks hooks install` opt-in).

## Performance

Audit scanner target: < 2s on a repo with 50+ markdown files. Strategy:
- Stream-read files (no full-text store in memory)
- Per-file regex (O(n) per file, n = file size)
- No recursive glob searches beyond 1 level (skills/*/SKILL.md, not skills/*/references/*.md)
- Optional: pre-compiled regex cache per call
