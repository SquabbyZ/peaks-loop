蜂群能力在 Peaks 里的合理位置

核心判断：蜂群不是产品主语，而是 Workflow 执行层的一种加速器。它应该在“需要并行判断/验证/产物生成”时被推荐和启用。

Peaks Workflow-first Architecture
User Intent
我要重构 / 做 UI / QA / ship / 调研
↓
Workflow Router
选择 peaks-code refactor/ui/qa/ship 等场景工作流
↓
Consultative Recommendation
推荐 skills / MCP / agents / profiles / 是否启用 swarm
↓
Single-agent path
低复杂度
低风险
Swarm path
并行扫描
并行评审
并行验证
External capability path
MCP / specialist skill
↓
CLI Execution + Evidence
dry-run · doctor · artifacts · rollback · report

我的建议

蜂群只在 Peaks 已经有清晰工作流和产物契约后介入；它提升吞吐，不替代决策。用户看到的是“推荐启用蜂群：预计并行处理这些子任务”，而不是直接暴露一堆 agent。
