## 1. What's in this release

- **zcode adapter** — peaks-loop's 9th supported IDE (`src/services/ide/adapters/zcode-adapter.ts`, 98 lines). z-code is a desktop application (no CLI binary); `compact.compactCommand` / `hookEvent` / `toolMatcher` / `envVar` are degraded with UNVERIFIED sentinel values, and `standardsProfile` borrows `CLAUDE.md` + `.claude/rules` paths since z-code reuses the upstream `.claude/` convention. New `IdeId` member + `IDE_DETECTION_DIRS` entry + `IDE_SKILL_INSTALL_PROFILES` entry.
- **`peaks ide model --current` CLI** — new `src/cli/commands/ide-commands.ts` (80 lines) + register in `src/cli/program.ts`. Outputs `{ modelId, detected, registeredAdapters }`. Available as a top-level primitive (not adapter-specific).
- **Runtime model detection service** — new `src/services/ide/current-model-detector.ts` (43 lines). Walks the adapter registry in insertion order, calls each adapter's optional `detectCurrentModel()` with try/catch isolation.
- **`IdeAdapter` interface optional field** — `readonly detectCurrentModel?: () => Promise<string | undefined>` in `src/services/ide/ide-types.ts`. Back-compat: 8 existing adapters without the field are unaffected.
- **`getStrongestModelIdAsync()` async variant** — `src/services/config/model-routing.ts`. Sync `getStrongestModelId()` unchanged; new async variant lets future async callers fall back to runtime probe instead of `'claude-opus-4-7'`.
- **Install no longer writes hardcoded default model** — `scripts/install-skills.mjs` removes `model: 'sonnet'` + `providers.minimax.model` defaults. `STRONGEST_MODEL_ID` constant removed, replaced by `getStrongestModelId(config?)` with 3-layer fallback (`config.model` → `PEAKS_STRONGEST_MODEL_DEFAULT` env var → back-compat `'claude-opus-4-7'`). 16 call sites in `workflow-router-service.ts` + `minimax-worker-service.ts` + `rd-service.ts` updated.
- **SKILL.md / CLI drift sync** — `skills/peaks-code/SKILL.md` updated with inline `> CLI reality check` blocks at §Step 0.8 / §Step 2.5 / §Step 11c/11d + new §CLI Drift Index section. `skills/peaks-slice-decompose/SKILL.md` gets a new §SC template hard rules section with 3 rules (ESM/CJS pre-flight, whitelist fixture sync as primary task, adapter-interface optional fields).
