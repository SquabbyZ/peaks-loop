## 8. Failure recovery (if the publish fails)

| Failure | Recovery |
|---|---|
| `npm ERR! 403 You do not have permission` | `npm login` as the user with publish rights for `peaks-loop` |
| `npm ERR! 409 Version already exists` | Someone else (probably the user) already published 4.0.0-beta.3. Bump to `4.0.0-beta.4`, re-run §3.2 + §3.3. |
| `npm ERR! 402 Payment required` | The `peaks-loop` package name is taken on npm. Unlikely; the user has shipped earlier betas (4.0.0-beta / 4.0.0-beta.2). If it happens, contact npm support. |
| `npm ERR! network` | Retry. The tarball is already built at `.pack-cache/peaks-loop-4.0.0-beta.3.tgz`. |
| README video does not render on npmjs.com | The card uses raw.githubusercontent.com URLs, which npmjs.com's renderer resolves. If npmjs.com's renderer strips the link, fall back to a `https://asciinema.org/...` or a YouTube embed (not implemented; out of scope for this release). |
