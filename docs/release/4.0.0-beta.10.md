## 0. What's in this release (high level)

4.0.0-beta.10 is a **CLI surface cleanup** + **on-demand ECC**
release. Three independent slices land in this beta:

1. **Slice 1 — `del-minimax-worker`** — pure delete of the historical
   MiniMax provider integration. 7 ACs.
2. **Slice 2 — `hide-role-skills`** — 10 role-skill CLI commands
   hidden from `--help` (still invokable internally). 8 ACs.
3. **Slice 3 — `peaks ecc install|status|ls|show`** — drop the
   subprocess model, give users a download + cache surface, let the
   LLM consume cached `agents/*.md` directly. 11 ACs.

**Merge order**: 1 → 2 → 3 (NOT parallel). Slice 3 flips
`peaks agent` default behavior; Slice 2 hides it from `--help` so
users don't see a "new behavior on a command they didn't know existed."

---

## 1. Why this release exists

User feedback (2026-07-15 session): audit of `peaks-loop` v4.0.0-beta.9
surfaced 3 categories of cruft + a missing integration with
affaan-m/everything-claude-code.

### 1.1 Slice 1 driver
- `peaks minimax-worker` is a historical integration that no SKILL.md
  references; clean removal improves install size + reduces test
  surface.

### 1.2 Slice 2 driver
- The **Enhancement, not new AI CLI** tenet (CLAUDE.md) + RL-8
  (locked 2026-07-08) say role-skill CLIs are LLM-internal.
  10 commands accidentally leak into user-facing `--help`.

### 1.3 Slice 3 driver + Gate S3-0
- Originally: `peaks agent run <name>` would spawn
  `<cached>/ecc agent run <name> --json`. RD sub-agent verified the
  affaan-m/ECC repository — **no `ecc` binary exists**. The real
  structure is `agents/*.md` flat + SKILL.md descriptors.
- User chose **Option B**: drop subprocess entirely. peaks-loop is
  download + cache only; LLM reads `agents/*.md` directly (Skill-first
  path). This honors the "Enhancement, not new AI CLI" tenet.
