## 1. What's in this release

This is a **hotfix** on top of 4.0.0-beta.8. The advertised beta.8 scope (project-scan bootstrap + monorepo fix) is unchanged. Beta.9 only fixes the runtime path layout so the published tarball actually works when consumed via `npm install peaks-loop`.

### Fixed — `npm install peaks-loop` runtime path layout

User feedback (2026-07-15, immediately after beta.8 publish): `npm i -g peaks-loop` succeeded but `peaks -v` failed with:

```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module
'.../node_modules/peaks-loop/dist/src/cli/index.js'
imported from .../node_modules/peaks-loop/bin/peaks.js
```

Root cause (3 layered defects):

1. `tsconfig.build.json` set `rootDir: src` but tsc walked cross-folder imports and emitted the runtime tree at `dist/cli/`, `dist/services/`, etc. — not `dist/src/cli/` etc.
2. `bin/peaks.js` still pointed at `'../dist/src/cli/index.js'` (an older assumption).
3. `package.json#files[]` whitelisted `dist/src/**/*.js` / `.d.ts` but **never** `.md` — so the 4 audit/business templates bundled by `copy-templates.mjs` were never included in the tarball.

Fixes in this commit:

- `bin/peaks.js`: `'../dist/src/cli/index.js'` → `'../dist/cli/index.js'`.
- `scripts/copy-templates.mjs`: dest join `dist/src/services/...` → `dist/services/...`.
- `package.json#files[]`:
  - `dist/src/cli/index.js` → `dist/cli/index.js`
  - `dist/src/**/*.js` → `dist/**/*.js`
  - `dist/src/**/*.d.ts` → `dist/**/*.d.ts`
  - **add `dist/**/*.md`** (NEW)

### Includes — beta.8 scope unchanged

Everything from 4.0.0-beta.8 is included. See `docs/release/4.0.0-beta.8.md` and `CHANGELOG.md` for the full beta.8 scope (project-scan bootstrap slice + monorepo fix).
