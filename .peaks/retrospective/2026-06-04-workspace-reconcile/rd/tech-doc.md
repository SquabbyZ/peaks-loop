## Goals

- Add `peaks workspace reconcile` CLI command that scans `.peaks/2026-MM-DD-session-*/` directories, identifies a canonical session, re-points `.peaks/.session.json`, and (optionally, with `--apply`) deletes empty/abandoned session dirs older than 7 days.
- Modify SC `validate` and `boundary` so the artifact lookup checks `.peaks/.active-skill.json` first, then `.peaks/.session.json`, then a `find .peaks/ -name '<artifact>'` fallback. Surface the resolved session id as `data.resolvedSessionId` (additive field).

## Non-goals

- No merging of PRD/RD/QA artifacts across sessions; only re-pointing bindings.
- No migration of the deprecated `b60252` session dir.
- No change to existing SC command output envelope shape beyond the additive `resolvedSessionId` field.
- No new top-level dependencies; hand-rolled with `node:fs`, `node:path`.
- No change to the destructive-`--apply` gate posture outside W3's new `--apply`.

## Architecture decisions

1. **Reconcile is its own service, not a method on `WorkspaceInit` service.** The existing `workspace-service.ts` owns `init` which creates new session dirs. Reconcile is the inverse (re-point existing dir, optionally delete). Keeping them separate preserves the invariant: `init` never deletes, `reconcile` may delete.

2. **Hand-rolled session discovery via `fs.readdirSync`.** The repo already uses `fs.readdirSync` with `withFileTypes` in `session-manager.ts:listSessionMetas`. We use the same pattern, filtering on the `^\d{4}-\d{2}-\d{2}-session-[a-f0-9]+$` regex.

3. **Canonical session heuristic (4 tiers, deterministic order):**
   - Tier 1: `.peaks/.active-skill.json` `sessionId` if it points to a real entry under `.peaks/`.
   - Tier 2: entry whose `session.json` has the most recent mtime.
   - Tier 3: entry whose any file has the most recent mtime.
   - Tier 4: entry whose dir name sorts last lexicographically (newest YYYY-MM-DD wins).

4. **Re-pointing uses the existing `setCurrentSessionBinding` helper from `session-manager.ts`.** This is the canonical writer of `.peaks/.session.json`; reuse avoids drift.

5. **Deletion candidate rule: mtime > `ageThresholdMs` AND `artifactCount === 0`** (where `session.json` itself does not count, since it's auto-generated by `init`). The 7-day default matches the PRD.

6. **`--apply` semantics: actually `rm -rf` the dir.** Use `fs.rmSync(..., { recursive: true, force: true })`. Report each deletion in the JSON envelope. If a deletion fails, record the failure in the envelope; do not throw — return the partial-success envelope so the user can see what landed.

7. **SC resolution helper lives in `sc-service.ts`, not `artifacts/workspace-service.ts`.** Avoids widening the public surface of the artifact service. The helper signature is internal to sc-service.

8. **Resolution precedence (3 tiers):**
   1. Read `.peaks/.active-skill.json`; if `sessionId` exists AND the slice id's expected artifact (e.g. `qa/test-reports/<rid>.md` or `prd/requests/<rid>.md`) lives under that session's dir, return it with `candidateSources: ['active-skill']`.
   2. Else read `.peaks/.session.json`; if that session has the artifact, return with `['active-skill', 'session-json']` (active-skill was checked but didn't have the artifact; session-json is the next source).
   3. Else `find .peaks/ -name '<artifact>'`; if found in any session dir, return that session id with `['active-skill', 'session-json', 'find-fallback']`.
   4. Else `{ resolvedSessionId: null, candidateSources: [] }`.

9. **`resolvedSessionId` is additive on both `validate` and `boundary` envelopes.** The legacy fields are unchanged. Consumers that strict-parse the envelope do not break.

## Data flow (W3 reconcile)

```
peaks workspace reconcile --json
  └─ resolveCanonicalProjectRoot(options.project)
  └─ reconcileWorkspace({ projectRoot, apply: false, olderThanMs: 7d-default })
       ├─ discoverSessions(projectRoot)
       │    └─ readdirSync(.peaks) → filter on session id regex
       │    └─ for each entry: stat session.json mtime + recursive file count
       ├─ read active-skill from .peaks/.active-skill.json
       ├─ pickCanonicalSession(entries, activeSkillSessionId)
       │    └─ tier 1: active-skill match
       │    └─ tier 2: latest session.json mtime
       │    └─ tier 3: latest any-file mtime
       │    └─ tier 4: lexicographic last
       ├─ getSessionId(projectRoot) → previous binding
       ├─ if canonical !== previous: setCurrentSessionBinding(projectRoot, canonical)
       │    └─ repointedFrom = previous, repointedTo = canonical
       ├─ findDeletionCandidates(entries, ageThresholdMs)
       │    └─ mtime older than threshold AND artifactCount === 0
       └─ applyDeletions(candidates, apply)
            └─ if apply=true: rmSync each dir, return { deleted, wouldDelete: [] }
            └─ if apply=false: return { deleted: [], wouldDelete }
       └─ return ReconcileResult envelope
```

## Data flow (W4 SC resolution)

```
peaks sc validate --slice-id <rid> --json
  └─ validateArtifactRetention(sliceId)
       ├─ workspace = getWorkspaceConfigForPath(cwd)
       ├─ resolveArtifactSession(workspace, sliceId, projectRoot)
       │    ├─ tier 1: read .peaks/.active-skill.json
       │    │    └─ if sessionId is set AND .peaks/<sid>/qa/test-reports/<rid>.md exists: return { resolvedSessionId, candidateSources: ['active-skill'] }
       │    ├─ tier 2: read .peaks/.session.json
       │    │    └─ if sessionId is set AND artifact exists in that session: return { resolvedSessionId, candidateSources: ['active-skill', 'session-json'] }
       │    ├─ tier 3: find .peaks/ -name '<expected-artifact-name>' (recurse)
       │    │    └─ if hit: return { resolvedSessionId: <dir-name>, candidateSources: ['active-skill', 'session-json', 'find-fallback'] }
       │    └─ else { resolvedSessionId: null, candidateSources: [] }
       ├─ if resolvedSessionId: changeDir = .peaks/<resolvedSessionId>
       │    └─ else: changeDir = .peaks/<sliceId> (legacy behavior)
       ├─ RETENTION_REQUIREMENTS: check each artifact against changeDir
       └─ return { valid, missingArtifacts, warnings, resolvedSessionId, candidateSources }
```

The same `resolveArtifactSession` is wired into `recordCommitBoundary` so `peaks sc boundary` also benefits.

## Test plan

### W3 unit tests (`tests/unit/workspace-reconcile-service.test.ts`)

1. `discoverSessions` returns empty array when `.peaks/` does not exist.
2. `discoverSessions` returns all `YYYY-MM-DD-session-XXXXXX` dirs, sorted by name.
3. `pickCanonicalSession`: tier 1 wins when active-skill sessionId is a real entry.
4. `pickCanonicalSession`: tier 2 wins when active-skill is null.
5. `pickCanonicalSession`: tier 3 wins when no session.json exists.
6. `pickCanonicalSession`: tier 4 wins as last resort.
7. `repointSessionJson`: writes new `.session.json`; `repointedFrom`/`repointedTo` correct.
8. `repointSessionJson`: no-op when already canonical (still sets lastActivity; repointedFrom === repointedTo).
9. `findDeletionCandidates`: empty dirs older than threshold are candidates.
10. `findDeletionCandidates`: dirs with artifacts are never candidates.
11. `findDeletionCandidates`: only `session.json` counts as auto-generated and does not disqualify.
12. `applyDeletions(apply: false)`: returns `wouldDelete`, no disk mutation.
13. `applyDeletions(apply: true)`: actually `rm -rf`s; returns `deleted`.
14. `reconcileWorkspace` top-level: end-to-end dry-run, all fields present, JSON envelope shape.

### W4 unit tests (extend `tests/unit/sc-service.test.ts`)

1. `validateArtifactRetention` returns `resolvedSessionId === active-skill sessionId` when the artifact is under that session dir.
2. Returns `resolvedSessionId === session.json sessionId` when active-skill doesn't have the artifact but session.json does.
3. Returns `resolvedSessionId === <fallback>` when only the find path hits.
