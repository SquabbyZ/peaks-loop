4.0.0-beta.5(发布日 2026-07-08,10 commits 增量,轴心 = **上线 peaks-solo dispatcher**)。

## 1. 增量表面 = 4 slice

| slice | 性质 | 产物 |
|---|---|---|
| **S0** | code | `peaks skill search` 顶层 primitive(`src/services/skill/skill-search-service.ts` + `src/cli/commands/skill-search-commands.ts`,Zod 校验,substring match,JSON 输出) |
| **S1** | skill | `peaks-solo` 分诊员 skill + 3 references + 7 单元测试 |
| **S2** | integration | `.claude-plugin/marketplace.json` 注册 + CHANGELOG 4.0.0-beta.5 段 + README/EN paragraph + ≤ 20 字 peaks-code SKILL.md 指针 |
| **S3** | dogfood | `tests/integration/dispatcher-flow.test.ts`(7 case T-1..T-7)+ GitHub top 10 端到端验证 + 4-section brief memory |

执行序 = `S0 ‖ S1 → S2 → S3`(S0/S1 并行,S2 等两者,S3 等 S2)。Mode:`full-auto` for S0/S1/S3,`assisted` for S2(CHANGELOG 措辞需 user 冻结,spec §1.1)。

## 2. 新增 skill / CLI

- **新增 skill** — `peaks-solo`(分诊员)。第三条独立入口,跟 `peaks-code / peaks-content / peaks-doctor` 平级。**不**持有 implementation surface,只看 / 查 / 转交 / 跑通用工具 / 提议沉淀;**不**写代码 / 不写 PRD / 不跑 vitest / 不改 Loop Engineering 资产。
- **新增 CLI** — `peaks skill search`(顶层 primitive,不只是 dispatcher 私用)。query / tag / domain 过滤,JSON 输出。
- **references 3 个** — `triage-decision-table.md` / `fallback-tool-inventory.md` / `sediment-prompt-template.md`。
- **老入口 0 改动** — `peaks-code / peaks-content / peaks-doctor / peaks-issue-fix-orchestrator / peaks-sop / peaks-resume / peaks-status / peaks-test` 全部向后兼容;`peaks-code` 11 步流程 unchanged(HC-8 / HC-10)。

## 3. 分诊流(HC-9 + HC-11 锁死)

```
NL → peaks skill search --query <NL> → candidates
  - 0 → 自规划兜底(§4)
  - 1 → 直接 Skill tool 调起
  - ≥2 → AskUserQuestion 多选(不允许自由文本)
  - 兜底跑完 → 必问沉淀(默认 a)
  - 转交 leaf → leaf 内部 Step 11 沉淀,不重复问
```

## 4. 自规划兜底工具清单

| allowed | blocked |
|---|---|
| `deep-search` / `WebSearch` / `WebFetch` / `Bash`(非 src/ 业务)/ `Edit / Write` markdown + yaml + `.peaks/memory/` | `src/**` 业务代码 / `skills/peaks-{code,content,doctor,...}/` 任何 leaf / `peaks asset crystallize` 主动跑 |

## 5. 沉淀 4-option AskUserQuestion

(a) 普通 lesson / convention → `peaks memory extract --apply`(默认推荐)  
(b) Loop Engineering 资产 → `peaks asset crystallize`(用户主权,LLM 不自动跑)  
(c) 改 scope 再沉淀  
(d) 不沉淀(单次,丢弃) — 不允许 LLM 默认推荐

**Why:** 这一版唯一的产品变化是 dispatcher 上线(S0/S1/S2/S3),其它一切(`peaks-code` 流程、schema、CLI verb、karpathy / darwin 红线)全部继承 beta.4。任何"4.0.0-beta.5 vs beta.4 差异"问题,**先排除 dispatcher 维度,再展开**,否则会陷入已经稳定的旧资产细节。
