Audit artifact writer generalization (Slice 2026-06-26-audit-artifact-writer-generalization). Replaces the narrow `peaks audit static --record` writer with a polymorphic 4-writer surface that covers every artifact type produced during a peaks-loop audit.

**Why:** On 2026-06-22 the Plan 4 audit produced 4 artifact types (audit prompts, machine-output JSON, narrative report, decision verdict) but the canonical writer (`src/services/audit/decision-writer.ts`) only knew how to persist one of them (`RedLineAudit` snapshot). The author was under release pressure and `git add`-ed the other 3 types directly into `.peaks/memory/` without going through any CLI, leaving 4 orphan files and breaking the shape contract. This slice generalizes the writer so all 4 types have a first-class CLI surface; hand-`git add` into `.peaks/memory/` is no longer the path of least resistance.

## The 4 writers

| Writer | CLI surface | Output path | Frontmatter `metadata.type` | `metadata.artifactType` |
|---|---|---|---|---|
| `writeDecision` | (existing) `peaks audit static --record` | `.peaks/memory/audit-decisions/<slug>.md` | `decision` | (legacy, no field) |
| `writePrompt` | `peaks audit artifact write --kind prompt` | `.peaks/memory/audit-prompts/<slug>.md` | `reference` | `prompt` |
| `writeMachineOutput` | `peaks audit artifact write --kind machine-output` | `.peaks/memory/<slug>.md` (top level) | `project` | `machine-output` |
| `writeNarrative` | `peaks audit artifact write --kind narrative` | `.peaks/memory/<slug>.md` (top level) | `project` | `narrative` |

**Slug pattern:** `<prefix>-<YYYY-MM-DD>[-<rid>]` where prefix is `audit-decision` / `audit-prompt` / `audit-output` / `audit-narrative`. Override with `--name` (used as slug verbatim).

## Schema invariants (enforced by writers + unit tests)

Every artifact file MUST have:
- `name:` (top-level, kebab/snake slug)
- `description:` (top-level, single line)
- `metadata.type:` ∈ `{project, rule, decision, reference, feedback, convention, module, lesson}` (the 8 `ProjectMemoryKind` values, enforced by `VALID_PROJECT_MEMORY_KINDS` exported from `project-memory-service.ts`)
- `metadata.artifactType:` ∈ `{decision, prompt, machine-output, narrative}` (4 choices; additive — `parseStoredMemoryFile` ignores unknown keys so it's safe to read by old tooling)
- `metadata.sourceArtifact:` (CLI provenance)
- `metadata.createdAt:` (ISO date)

`machine-output` writer additionally: validates input is JSON-parseable, embeds the raw JSON **byte-for-byte** inside a fenced ` ```json ` block (no reformatting, no key reordering). The envelope adds an H1 + summary table for human readers.

## Directory whitelist (enforced by `tests/unit/workspace/memory-shape-guard.test.ts`)

`.peaks/memory/` may contain ONLY:
- `index.json` (the memory index; regenerated by the memory service)
- `<slug>.md` files (top level) with canonical frontmatter
- `audit-decisions/<slug>.md` (decision artifacts)
- `audit-prompts/<slug>.md` (methodology references)

Anything else — a `.json` at top level, a new subdirectory, an unfrontmatter'd `.md` — fails the guard (HARD) or shows up as a SOFT warning for legacy cleanup.

## How to apply

1. **Adding a new artifact during a peaks-loop audit** — pick the closest kind from the 4-table above. Use the corresponding CLI flag. NEVER hand-write files into `.peaks/memory/`.
2. **Reading artifacts** — `peaks project memories --kind decision | --kind reference | --kind project` filters by `metadata.type`. `peaks memory search "..."` does fuzzy match across all frontmatter fields including `artifactType`.
3. **Migrating legacy non-conformant files** — read the file, pick the matching writer, call it with the same `slugOverride` to preserve the original filename, delete the original. See migration history in this slice's commit for the exact recipe.

## When the guard fails in CI

`tests/unit/workspace/memory-shape-guard.test.ts` fails on HARD violations:
- A `.json` file other than `index.json` was committed at the top of `.peaks/memory/`
- A new top-level subdirectory was created (other than `audit-decisions/` / `audit-prompts/`)

The error output names the exact path. Fix: route the new write through `peaks audit artifact write --kind <matching>` and remove the raw file from git.
