Under `.peaks/`, any top-level directory whose name starts with `_` is **ephemeral** and is automatically gitignored by the `.peaks/_*/` rule in `.gitignore`. Non-underscored top-level segments are **git-tracked** (source-of-truth artifacts).

**Why:** 2.7.1 collapsed the workspace to a single axis (`.peaks/_runtime/<sessionId>/` for session state). Before this convention, `.gitignore` accumulated 10+ ad-hoc rules (`.peaks/_runtime/`, `.peaks/_dogfood/`, `.peaks/_sub_agents/`, `.peaks/audit/`, `.peaks/system/`, `.peaks/2026-*-*/`, ...) that drifted as the layout evolved. The single `.peaks/_*/` rule is self-documenting: if a top-level dir is ephemeral, **rename it with a leading underscore**. The underscore is the convention; `.gitignore` derives the rest.

**How to apply:**
1. **Adding a new ephemeral top-level dir under `.peaks/`** → name it with a leading underscore (e.g. `_cache/`, `_preferences.json`). It is gitignored automatically. No `.gitignore` edit required.
2. **Adding a new git-tracked top-level dir** → no underscore prefix. It is tracked. Document the intent (what lives here, who owns it) in a memory under `.peaks/memory/`.
3. **Two explicit exceptions** live in `.gitignore` (each with a one-line justification comment):
   - `.peaks/runtime/` (no underscore) — typo guard for accidental writes to the wrong path
   - `.peaks/preferences.json` (no underscore) — per-project user state, regenerated by `preferences-service`. Could be moved to `.peaks/_preferences.json` but requires touching 13 src files; out of scope for the 2.8.0 slice.
4. **Migration of pre-convention dirs** (e.g. legacy `audit/`, `system/`, `2026-*-*/`) → either rename to `_*` form and let the wildcard catch them, or delete the dir if it was already empty. Do NOT add explicit `.gitignore` rules for non-underscored ephemeral dirs — that re-introduces the drift this rule is meant to prevent.

**Validation:** `git check-ignore` should report `true` for `.peaks/_runtime/foo`, `.peaks/runtime/foo`, and `.peaks/preferences.json`; `false` for `.peaks/PROJECT.md`, `.peaks/memory/foo.md`.
